# 小程序登录态持久化排查记录

日期：2026-04-28

## 问题描述

Taro 小程序登录成功后，刷新或重新进入小程序，页面又表现为未登录，需要重新点击“我的”页登录按钮。现象类似登录 token 没有像 Cookie 一样持久化。

## 旧项目实现对比

旧项目路径：`D:\student-engineering\student-manager-moblie`

关键实现：

- `src\App.vue`
  - 启动时从本地缓存恢复：
    - `token: uni.getStorageSync("token")`
    - `userId: uni.getStorageSync("userId")`
- `src\pages\mine\index.vue`
  - 登录成功后写入本地缓存：
    - `uni.setStorageSync("token", token)`
    - `uni.setStorageSync("userId", id)`
  - 同时写入 `getApp().globalData.token`
- 其他页面通过 `getApp().globalData` 取 `token/userId` 请求接口。

结论：旧项目方案本身是可行的，核心是“登录成功写 storage，App 启动从 storage 恢复到 globalData”。

## 会员福利详情状态逻辑

旧项目会员福利详情不是只请求卡片详情，还会额外请求当前用户的卡片领取/使用状态：

- 会员卡详情：`src\pages\welfare\detail.vue`
  - 请求详情：`app/vip/info/{cardId}`
  - 请求状态：`app/card/info/{cardId}/{userId}/{type}`
- 我的卡包详情：`src\pages\cardpackage\detail.vue`
  - 请求详情：`app/vip/info/{cardId}` 或 `app/welfare/info/{cardId}`
  - 请求状态：`app/card/info/{cardId}/{userId}/{type}`

该状态接口决定底部按钮显示：

- 无登录态：显示“点击领取”，跳转登录。
- 有登录态但无领取记录：显示“点击领取”。
- `status == '2'`：显示“点击领取”。
- `status == '1'` 且会员卡：显示“已领取”。
- `status == '1'` 且普惠福利 `useStatus == '1'`：显示“点击使用”。
- `status == '1'` 且普惠福利 `useStatus == '2'`：显示“已使用”。

Taro 版已按该思路处理：

- 详情页加载时先请求 `app/{vip|welfare}/info/{cardId}`。
- 如果本地缓存能恢复 `userId`，再请求 `app/card/info/{cardId}/{userId}/{type}`。
- 如果状态接口没有记录，底部仍显示“点击领取”。
- 领取时调用 `POST app/card/receive`，由后端按 `cardId + userId + type` 创建或更新卡片记录，避免旧逻辑中没有 `status.id` 导致无法领取。

注意：当前请求封装中，当后端返回 `data: null` 或无 `data` 时会返回 `true`。详情页状态处理必须排除 `true/false`，否则会误认为已有状态对象，导致底部按钮消失。

## Taro 项目当前修正方案

项目路径：`D:\student-engineering\student-manager-taro`

已改为统一登录态缓存：

- `src/utils/app.ts`
  - 新增 `auth` 缓存对象：
    - `token`
    - `refreshToken`
    - `userId`
  - `saveAuthStorage()`：登录成功或刷新 token 后统一写入 `auth`，并兼容写入旧 key。
  - `readAuthStorage()`：优先读取 `auth`，兼容旧 key。
  - `refreshAuthFromStorage()`：从 storage 恢复到 `globalData`。
  - `clearAuthStorage()`：退出登录时清理 `auth/token/refreshToken/userId`。
- `src/app.tsx`
  - 小程序启动时用 `readAuthStorage()` 恢复 `globalData.token/userId`。
- `src/utils/request.ts`
  - 请求 token 优先从 `readAuthStorage()` 获取。
  - refresh token 成功后调用 `saveAuthStorage()` 写回本地缓存。
- `src/pages/mine/index.tsx`
  - 登录成功后调用 `saveAuthStorage()`。
  - 页面显示时通过 `refreshAuthFromStorage()` 恢复登录态。

## 仍需验证的点

1. 登录成功后，在微信开发者工具 `AppData / Storage` 中确认存在：
   - `auth`
   - `token`
   - `refreshToken`
   - `userId`
2. 点击开发者工具“编译/刷新”后，确认 storage 没有被清空。
3. 重新进入“我的”页，确认不再出现登录按钮。
4. 如果刷新后 storage 为空，优先检查开发者工具是否启用了清缓存、是否切换了 AppID 或项目目录。
5. 如果 storage 仍在但页面未登录，检查 `AppData` 里的 `globalData.token/userId` 是否恢复成功。

## 注意事项

- 微信小程序没有浏览器 Cookie 登录模型，应使用 `wx.setStorageSync / Taro.setStorageSync` 持久化 token。
- `globalData` 只存在于当前小程序进程内，刷新或重启后必须从 storage 恢复。
- 不能只依赖页面内 React state 判断登录，否则页面重挂载后会表现为未登录。
- 开发者工具清缓存、切换 AppID、重新导入项目，都可能清掉本地 storage。

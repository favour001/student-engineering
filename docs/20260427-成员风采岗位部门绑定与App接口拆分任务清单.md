# 成员风采岗位部门绑定与 App 接口拆分任务清单

作者：lin

## 任务清单

| 序号 | 任务 | 完成情况 |
| --- | --- | --- |
| 1 | 成员风采补系统岗位、系统部门绑定字段 | 已完成 |
| 2 | 后台成员风采新增/编辑支持选择岗位和部门 | 已完成 |
| 3 | 新增小程序成员风采领域 App 接口 | 已完成 |
| 4 | Taro 成员风采接入新 App 接口并恢复岗位/部门筛选 | 已完成 |
| 5 | 构建验证 | 已完成 |

## 完成过程中遇到的错误

| 问题 | 原因 | 修复 |
| --- | --- | --- |
| 成员风采原数据缺少结构化岗位/部门 | `lx_member_style` 只有展示字段，没有 `post_id/dept_id` | 新增字段和迁移 |
| Taro 曾直接请求后台管理接口 | `student-business/member` 是后台 CMS 接口，不适合作为小程序接口 | 新增 `/app/member-style/*` 小程序领域接口 |
| Web 构建找不到岗位/部门 API | 相对路径多退了一层 | 修正为 `../../platform/...` |
| 后端构建 Map 类型推断失败 | `new Map(posts.map(...))` 推断为 `Map<unknown, unknown>` | 显式声明 `Map<number, SysPost/SysDepartment>` |

## 排查记录

| 排查项 | 结果 |
| --- | --- |
| 系统岗位 | 已有 `sys_post`，可作为业务基础数据复用 |
| 系统部门 | 已有 `sys_department`，可作为业务基础数据复用 |
| 成员风采表 | `lx_member_style` 缺少 `post_id/dept_id` |
| 后台成员风采接口 | `student-business/member?category=member-style` 已存在，适合后台管理 |
| 小程序接口 | 原 `app/user/list` 命名不清晰，已新增 `app/member-style` 领域接口 |

## 修复方案

| 模块 | 方案 |
| --- | --- |
| 数据库 | `lx_member_style` 增加 `post_id`、`dept_id` 和组合索引 |
| 后端后台业务 | `MemberBusinessService` 创建、更新、列表筛选、映射均支持 `postId/deptId` |
| 后端 App 接口 | 新增 `AppMemberStyleController` 和 `AppMemberStyleService` |
| 后台 Web | 成员风采表单加载系统岗位/部门，保存 `postId/deptId` |
| Taro | 调用 `app/member-style/list`、`app/member-style/posts`、`app/member-style/departments`、`app/member-style/:id` |

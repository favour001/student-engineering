## 完成过程中遇到的错误

| 问题 | 原因 | 修复 |
| --- | --- | --- |
| 底部导航激活态仍偏紫色 | 样式中残留紫色透明背景和阴影 | 改为橙色高亮背景、橙色阴影和橙色文字 |
| 底部导航固定层覆盖范围不自然 | 外层固定容器从 `top: 0` 开始覆盖全屏 | 改为底部固定容器并限制高度 |
| 文章打开公众号链接时返回栈异常 | 先进入文章详情页，再 `navigateTo` 到 WebView | 列表有外链时直接进 WebView，详情页兜底改为 `redirectTo` |

## 排查记录

| 排查项 | 结果 |
| --- | --- |
| 底部导航组件 | `CustomTabBar.scss` 仍存在紫色激活背景和阴影 |
| 文章列表点击 | `home/index.tsx` 统一跳转 `/pages/article/index?id=...` |
| 文章详情页 | 发现外链后使用 `navigateTo` 打开 `/pages/oAArticle/index`，导致返回时回到中间页 |
| 紫色残留 | `Tabs.scss`、`CardList.scss` 仍有紫色阴影 |

## 修复方案

| 模块 | 方案 |
| --- | --- |
| 底部导航栏 | 激活态改为橙色背景和橙色文字，外层 fixed 改为底部固定 |
| 全局样式 | 主按钮、标签阴影改为橙色系 |
| 首页入口 | 文章列表优先读取 `articleUrl`，是 URL 时直接进入 WebView |
| 文章详情兜底 | 外链文章使用 `redirectTo` 进入 WebView，避免返回栈保留中间页 |

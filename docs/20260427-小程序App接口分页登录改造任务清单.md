# 小程序 App 接口分页登录改造任务清单

作者：lin

## 任务背景

当前 `student-manager-taro` 仍沿用旧移动端的 `app/...` 接口路径，但 `student-manager-server` 主要提供后台管理接口，例如 `student-business/content`、`student-business/member`、`student-business/benefit`、`student-business/activity`。小程序端需要移动端专用接口，重点解决列表分页、微信手机号登录、登录态 7 天失效、登录入口体验等问题。

## 当前判断

| 项目 | 判断 |
| --- | --- |
| 是否需要小程序专用 `app` 接口 | 需要。Taro 和旧移动端都依赖 `app/...`，后端目前缺少这套面向小程序的接口层 |
| 列表是否需要分页 | 需要。小程序端不适合一次性加载所有数据，活动、公告、视频、成员风采、卡券等都应分页 |
| 登录是否应使用手机号 | 是。后台可预录入微信用户资料，手机号是主要匹配字段 |
| 是否需要保存微信标识 | 需要。登录成功后应保存或更新 `lx_wxuser.wxopenid` |
| token 是否需要 7 天失效 | 需要。refresh token 建议 7 天，过期后重新微信登录 |
| 是否需要单独登录页 | 暂不建议。优先在“我的”页做登录状态区和微信手机号一键登录按钮 |

| 时间 | 错误/问题 | 影响 | 状态 |
| --- | --- | --- | --- |

| 2026-04-27 | Taro 多个列表接口无分页参数 | 数据量增长后移动端会卡顿 | 待处理 |
| 2026-04-27 | 当前 Taro 登录只传 `Taro.login()` 的 code | 缺少手机号授权，无法按手机号匹配后台预录入用户 | 待处理 |
| 2026-04-27 | Nest POST 默认响应业务码为 201 | Taro `commonRequest` 只认 `code === 200`，会误判登录失败 | 已修复：小程序 POST 接口加 `@HttpCode(200)` |
| 2026-04-27 | 后端 `.env` 暂未配置 `WECHAT_APPID/WECHAT_SECRET` | 真实微信手机号登录无法完成联调 | 待配置 |
| 2026-04-27 | 分页请求失败可能导致 loading 锁死 | 后续无法继续触发上拉加载 | 已修复：分页加载函数使用 `try/finally` 释放 loading |
| 2026-04-27 | Taro 仍请求旧 `https://sdsosa.com/nsx-api/...` | 无法命中新建的 NestJS `app` 接口 | 已修复：请求封装改为 NestJS `/api` 前缀 |
| 2026-04-27 | 小程序产物中保留 `process.env.TARO_APP_API_*` | 小程序运行时可能没有 `process` 对象 | 已修复：使用 Taro `defineConstants` 编译期注入 |
| 2026-04-27 | Taro 图片地址只处理旧 `/nsx-api/image` | 后台上传返回的 `/image`、历史 `/home`、富文本 `<img>` 可能无法显示 | 已修复：参考 Web 的 `resolveAssetUrl` 规则统一处理 |

## 排查记录

| 排查项 | 结果 |
| --- | --- |
| Taro 接口扫描 | 发现大量 `app/...` 调用，包括文章、公告、活动、卡券、微信用户、登录等 |
| 旧移动端接口扫描 | 旧项目使用 `app/wxlogin/login`，传入 `phoneCode` 和 `loginCode` |
| 后端接口扫描 | 现有接口主要是后台管理的 `student-business/...`，未发现 `app/...` 小程序接口 |
| 微信用户实体检查 | `lx_wxuser` 已有 `mobile`、`wxopenid` 字段，可支撑手机号匹配和 openid 绑定 |
| token 机制检查 | 后台已有 JWT 和 7 天 refresh token，但绑定的是 `SysUser`，不适合直接复用给微信用户 |
| 第一轮实现 | 新增 `src/app-api` 小程序接口层，覆盖 Taro 当前主要 `app/...` 调用 |
| 登录实现 | Taro “我的”页改为 `openType="getPhoneNumber"`，后端接收 `phoneCode + loginCode` |
| 分页实现 | Taro 主要列表页统一传 `pageNum/pageSize`，使用 `useReachBottom` 加载下一页 |
| 接口切换 | NestJS 设置全局前缀 `api`，Taro 请求封装改为 `http://127.0.0.1:8888/api` |
| 图片显示 | 参考 Web 项目：`/image`、`/uploads`、`/files`、`/home`、相对路径和富文本 `img src` 统一转换为小程序可访问 URL |

## 修复方案

| 模块 | 方案 |
| --- | --- |
| 后端 App 接口 | 新增小程序专用接口层，路径保持 `app/...`，内部复用现有业务实体和部分 service 查询逻辑 |
| 分页 | 列表统一支持 `pageNum/pageSize`，返回 `list/total/pageNum/pageSize/hasMore` |
| 登录 | 小程序端使用 `openType="getPhoneNumber"` 获取 `phoneCode`，同时调用 `Taro.login()` 获取 `loginCode` |
| 微信接口 | 后端用 `loginCode` 换 `openid/session_key`，用 `phoneCode` 获取手机号 |
| 用户匹配 | 优先用手机号匹配 `lx_wxuser.mobile`，匹配成功后写入 `wxopenid` |
| 登录态 | 新增微信用户 token 存储或等效机制，access token 短效，refresh token 7 天 |
| 前端体验 | “我的”页展示登录按钮、用户信息、未录入手机号提示；需要登录的业务跳转到 `pages/index/index?tab=mine` |

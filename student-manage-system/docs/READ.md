# 前端文档规范

## 命名规则
1. 文档文件名必须使用：`YYYYMMDD-需求名称.md`
2. 文件名必须直接体现日期和需求，不允许再使用 `READ.md`、`WORKLIST.md` 作为实际任务文档名称。
3. `READ.md` 仅作为规范说明文件存在，不承载具体需求交付内容。

## 内容结构
具体需求文档必须至少包含以下表格化内容：
1. 需求信息表
2. 工作清单与完成情况表
3. 功能范围与边界表
4. 不做事项表
5. 验证测试表
6. 风险与后续计划表

## 编写要求
1. 日期统一使用绝对日期，格式为 `YYYY-MM-DD`。
2. 工作项必须写明“做什么、完成状态、影响范围、备注”。
3. 边界必须写清楚哪些功能属于本次范围，哪些不属于。
4. “不能做”必须明确列出当前版本不支持的内容，避免歧义。
5. 验证测试必须写明命令、结果、结论，不能只写“已验证”。
6. 需求文档要优先使用表格，不要只写散文描述。
